Modulul 4 - Lecția 8 - Grafica vectorială

Grafica vectorială

SVG (Scalable Vector Graphics) – un tip de format grafic și limbaj pentru descrierea imaginilor vectoriale. În interior, fișierul nu este binar, ci text simplu (XML) care descrie obiectele, efectele și comportamentul acestora. Grafica vectorială are o serie de avantaje.

  • Scalarea – spre deosebire de grafica bitmap, elementul SVG nu își pierde din calitate atunci când este scalat.
  • Dimensiunea fișierului – imaginile vectoriale sunt mult mai mici decât cele bitmap atunci dacă sunt folosite pentru iconițe sau alt tip de grafică abstractă. În schimb, utilizarea vectorilor pentru imagini fotorealiste are ca rezultat dimensiuni uriașe ale fișierelor.
  • Adaptare – folosind CSS și JavaScript, puteți modifica parametrii unei imagini vectoriale, cum ar fi culoarea sau marginile.

Elementele SVG

Orice imagine vectorială e compusă dintr-un set de forme care sunt reprezentate sub forma unor elemente SVG. Când un designer exportă o iconiță dintr-un editor grafic, cum ar fi Adobe Illustrator sau Inkscape, programul convertește imaginea într-un șablon și produce un fișier SVG. Developerul folosește imaginea SVG gata făcută fără a se gândi la conținutul acesteia, însă nu va fi de prisos o familiarizare de bază cu elementele, așa că haideți să analizăm unele dintre ele.

  • Elementul (svg)

    Acesta specifică pânza pentru desenarea unor forme. Haideți să facem o pânză de 400x300 pixeli și să adăugăm un stil de tip inline cu un outline pentru a desena bordurile.

    Exemplu :

    index.html

    HTML

    <svg width="400" height="300"
    style="outline: 2px dashed #2a2a2a">
    </svg>

  • Elementul (rect)

    Creează o suprafață dreptunghiulară. În continuare, vom desena un dreptunghi de 250x150 pixeli, căruia îi vom adăuga următoarele caracteristici.

    • Umplem suprafața cu culoare, folosind atributul fill.
    • Setăm culoarea neagră pentru bordură și o lățime de 4px , folosind atributele stroke și stroke-width.
    • Setăm deplasarea la 20px pe verticală și pe orizontală în raport cu colțul din stânga sus al pânzei cu atributele x și y.
    • Adăugăm rotunjirea colțurilor dreptunghiului cu atributele rx (orizontală) și ry (verticală). Dacă se specifică o singură valoare, cealaltă va fi identică.

    index.html

    (svg width="400" height="300" style="outline: 1px solid black") (rect width="250" height="150" fill="tomato" stroke="black" stroke-width="4" x="20" y="20" rx="40" /) (svg/)

  • Elementul (polygon)

    Descrie un poligon al cărui aspect este determinat de atributul points – un set de puncte sub formă de coordonate de linii, care sunt împărțite în grupuri.

    (svg width="600" height="300" style="outline: 2px dashed
    #2a2a2a")
    (polygon
    points="20,20 20,200 200,200 200,20"
    fill="orange"
    stroke="#2a2a2a"
    stroke-width="4"
    /)
    (/svg)

    Prima cifră a grupului este coordonata x, a doua cifră după virgulă este coordonata y. Primul grup de numere determină coordonatele începutului primei linii, al doilea grup determină sfârșitul primei linii și în același timp, începutul celei de-a doua dintre acestea și așa mai departe.

    Poligonul este o formă închisă, astfel încât ultima linie de închidere este adăugată în mod automat, conectând sfârșitul ultimei linii cu începutul primei linii al formei.

    Exemplu :

    HTML

    width="600"
    height="300"
    style="outline: 2px dashed #2a2a2a"

    (polygon

    stroke-width="4"

    stroke-width="4"

    (/svg)

Fișierul SVG

Grafica vectorială este stocată în fișiere cu extensia .svg. Acestea sunt fișiere "text simplu" care conțin la nivelul superior tag-ul (svg) cu un set de atribute de serviciu care indică în ce versiune a limbajului de marcare și în funcție de ce versiune a specificației SVG a fost scris fișierul. Dintre toate atributele de serviciu, doar atributul xmlns este obligatoriu, celelalte sunt de obicei omise.

(svg
version="1.1"
xmlns="http://www.w3.org/2000/svg"
xmlns:xlink="http://www.w3.org/1999/xlink"
xmlns:ev="http://www.w3.org/2001/xml-events"
)
(circle r="75" cx="50%" cy="50%" fill="tomato" /)
(/svg)

Dacă salvați acest cod într-un fișier extern, cum ar fi icon.svg, și îl adăugați la o pagină web folosind tag-ul , browserul va desena o pânză de 300x150 pixeli cu un cerc în centru.

INTERESANT

Dacă nu setați în mod explicit dimensiunea, browserul va seta elementului o valoare de 300x150 pixeli în mod implicit. Orice nu se potrivește va fi tăiat.

  • SVG-viewport

    Conținutul unui document SVG este desenat pe o pânză infinită și poate fi atât de mare cât doriți, dar partea vizibilă a pânzei corespunde dimensiunilor elementului (svg). Atributele width și height stabilesc dimensiunea ferestrei de vizualizare (viewport). Colțul din stânga sus al ferestrei de vizualizare este originea, unde x = 0 și y = 0. Valoarea implicită este reprezentată de pixeli, deci specificarea px este inutilă.

    (svg
    xmlns="http://www.w3.org/2000/svg"
    width="400"
    height="200"
    style="outline: 2px dashed #2a2a2a"
    )
    (circle r="75" cx="50%" cy="50%" fill="tomato" /)
    (/svg)

  • SVG-viewBox

    Printr-o simplă traducere a "view box", înțelegem că semnifică literalmente o "fereastră de vizualizare". Adică, aceasta este o "fereastră” prin care puteți vizualiza o parte a fișierului SVG. Este ca o fereastră a unui avion prin care se vedea lumea din jur, dar viewBox, de asemenea, poate scala imaginea (mărește sau micșorează) ca un telescop.

    Atributul viewBox specifică ce parte a imaginii va fi afișată și la ce scară.

    viewBox="x y width height"

    Valorile x și y controlează deplasarea față de coordonatele de început ale ferestrei de vizualizare (viewbox) și permit afișarea doar a unei părți a imaginii. Valorile width și height controlează scalarea.

    Dacă raportul atributelor width și height față de omologii lor, viewBox, este de 1:1, imaginea va fi redată la dimensiunea inițială.

    (svg width="400" height="200" viewBox="0 0 400 200")
    (circle r="75" cx="50%" cy="50%" fill="tomato" /)
    (/svg)

    Cu un raport de 2:1, imaginea originală va fi scalată exact de două ori, dar dimensiunea elementului va rămâne aceeași – 400x200.

    (svg width="400" height="200" viewBox="0 0 200 100")
    (circle r="75" cx="50%" cy="50%" fill="tomato" /)
    (/svg)

    Cu un raport de 1:2, imaginea originală va fi redusă exact de două ori, dar dimensiunea elementului va rămâne aceeași – 400x200.

    (svg width="400" height="200" viewBox="0 0 800 400")
    (circle r="75" cx="50%" cy="50%" fill="tomato" /)
    (/svg)

    ATENȚIE

    Adică, viewBox este o "fereastră" prin care utilizatorul se uită la fișierul SVG. Aceasta poate fi mutată în orice loc din fișierul SVG, iar utilizatorul va vedea fragmentul care se află direct sub viewBox, în funcție de scara selectată.

Utilizare

Există mai multe moduri de a folosi grafica vectorială, fiecare cu avantajele și dezavantajele sale.

    • Codul SVG încorporat mărește dimensiunea fișierului HTML.
    • Nu este posibilă reutilizarea aceleiași imagini fără a duplica codul.
    • Întreținerea și editarea de tip inline SVG necesită o muncă intensivă.
    • Spre deosebire de fișierele SVG externe, browserul nu poate stoca în cache formele de tip inline.

  • Sprite SVG

    Sprite SVG este o tehnică modernă care este folosită în cele mai multe cazuri.

    • Un fișier pentru toate iconițele SVG a unui site.
    • Fișierul sprite este stocat în cache de browser și nu va fi încărcat în cazul vizitelor repetate.
    • Prin CSS puteți schimba culoarea și alte caracteristici ale iconițelor dintr-un sprite.

Stilizarea figurilor

CSS vă permite să schimbați în mod dinamic unele dintre caracteristicile unei imagini vectoriale sau iconițe de tip inline dintr-un sprite SVG, cum ar fi tipul de umplere, culoare sau contur. Valorile pot fi setate atât prin atributele formelor SVG, cât și prin proprietățile CSS cu același nume. În practică, pentru stilizarea figurilor nu sunt folosite atributele, ci codul CSS, deoarece este mai ușor de întreținut și schimbat.

  • Proprietatea fill

  • Proprietatea fill-opacity

  • Proprietatea stroke

  • Proprietatea stroke-width

  • Proprietatea stroke-opacity
